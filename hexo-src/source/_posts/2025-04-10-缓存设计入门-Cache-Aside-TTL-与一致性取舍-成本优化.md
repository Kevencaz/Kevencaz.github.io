## 目标与指标

- 目标：缩短响应路径，降低等待时间
- 衡量口径：P95/INP/TTFB
- 复盘节奏：每次发布后复盘一次

## 方案路径

- 读：先查缓存，未命中再查数据库并回填。
- 写：先写库，再删缓存（或延迟双删）。
- 用 TTL + 版本号兜底，避免脏数据无限期存在。

## 深度推进

- 跨团队协作规范
- 多环境治理
- 成本与性能平衡

## 小型案例

下面的片段可作为最小闭环示例：

~~~js
const key = `user:${id}`;
let data = await cache.get(key);
if (!data) {
  data = await db.getUser(id);
  await cache.set(key, data, { ttl: 300 });
}
return data;
~~~

## 取舍与边界

- 如果只追求性能，可能带来新的风险：只优化局部导致瓶颈迁移
- 把复杂度锁在工具或中间层，避免侵入业务核心
- 优先保证可回滚，再谈极致优化
- 避免无边界扩张
- 关注长期维护成本

## 落地节奏

- 建议节奏：紧急修复热更新
- 先小范围灰度，再扩大覆盖
- 每一步都要有可观察数据
